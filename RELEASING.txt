0. Start on branch 'next_version'; when changes are ready to be released:
   git commit -a -m "<What's Changed?>"
   git checkout master
   git merge next_version

1. Update __version__ in soco_cli/__init__.py
2. Commit: git commit -a -m "Commit message"
3. Push: git push
4. Tag the commit: git tag -a vX.Y.Z -m "Version X.Y.Z"
5. Push the commit tags: git push --tags
6. Build: make clean; make build
7. PyPi check: make pypi_check
8. PyPi push: make pypi_upload

9. git checkout next_version
   git merge master
   git push
   start next set of changes
